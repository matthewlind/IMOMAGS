Descriptiontop

With this plugin you can display news from one or more RSS or Atom feeds. The news can be displayed with one of the 4 views I've included, or you can create your own view so you can customize how the news items are shown.

  • Merge up to 6 feeds in one news widget
  • View based so very easy to change the html structure and the data to show
  • Show newest items first
  • Randomize the news items
  • Number of news items to show
  • Use ajax to refresh the news widget every x seconds
  • RSS/Atom feeds are cached so they don't need to be redownloaded on every page load
  • Supports multiple instances
  • 4 example views included
  • Supports images

How to install top

Extract the zip you downloaded from codecanyon. Inside that zip you'll find a file called news-widget.zip. This is the widget file you need to install.
To do this go to Plugins > Add New, and select the Upload tab. Now upload and activate the news-widget.zip. After installation and activation you would have an extra widget in your Appearance > Widgets page:

How to use top

After the installation you can drag the widget to the location were you want to place it in your theme. Configure the settings and you are ready to go. All the settings are pretty simple and for the not so obvious ones I've added a description under the field.

As source for the news items you can use RSS and Atom feeds!

Create a custom view top

The views are located in the wp-content/plugins/news-widget/views/ directory. All views end with a suffix -view.php. So if you want to make your own view create a new file or copy and paste an existing view and rename it to [name-of-choise-here]-view.php. So for example my-view.php.

Below you find a list of variable that you can use in your view:

  • $before_widget: Widget prefix
  • $after_widget: Widget suffix
  • $title: Widget title
  • $before_title: Title prefix
  • $after_title: Title suffix
  • $entires: Is an array of feed items.

Below you find a list of properties that you can access on the news items you find in the $entries array.

  • title: The title of the news item
  • link: Link to the news items
  • author: Who has written this news item
  • publishedDate: Date when the news was published. If you want to customize the look of this date you can use the PHP date function and use it as so: date('Y-m-d', strtotime($entry->publishedDate))
  • contentSnippet: A small part of the news item
  • content: Full content of the news item
  • categories: An array of categories where this news is posted under
  • websiteTitle: The name of the website, for example 'CodeCanyon'
  • feedUrl: The original url from which feed this news is taken
  • websiteUrl: The url to the website
  • websiteDescription: A short description of the website
  • Image: If there is an image used in the content this property will contain the url to the first image
<?php echo $before_widget ?>
<?php if ( !empty( $title ) ) { echo $before_title . $title . $after_title; }; ?>
         
<!-- Widget Content start -->
<ul>
<?php foreach($entries as $entry): ?>
		<li>
		<h4><a href="<?php echo $entry->link; ?>"><?php echo $entry->title ?></a></h4>
			<p><?php echo $entry->contentSnippet ?></p>
</li>
<?php endforeach; ?>
</ul>
<!-- Widget Content end -->
         
<?php echo $after_widget ?>
         

Shortcode top

This widget has also support for shortcodes. This way you can also use it in your posts and pages. Below you find an overview of the different parameters that are possible.

  • title: The title to show above the widget
  • feed1: RSS/Atom feed url
  • feed2: RSS/Atom feed url
  • feed3: RSS/Atom feed url
  • feed4: RSS/Atom feed url
  • feed5: RSS/Atom feed url
  • feed6: RSS/Atom feed url
  • view: The view file you want to use. For exmaple basic-view
  • number: Number of items to show
  • cache: Number of seconds to cache the feeds
  • use_ajax: Set to true if you want to use the ajax refresh feature
  • random: Set to true if you want to show random items
  • ajax_refresh_time: Number of seconds to wait for a refresh if you have set use_ajax to true.
  • before_title: The element to place before the widget title. By default this is <h3>
  • after_title: The element to place after the widget title. Bydefault this is </h3>

Below you find an example of a shortcode:

[newswidget feed1="http://codecanyon.net/feeds/new-codecanyon-items.atom" view="basic-view" number="10" cache="60" use_ajax="true" random="true" ajax_refresh_time="20"]

I've also added an little feed button to insert and example news widget shortcode in your post. Then you can easily modify it to fit your needs.
